# Clinical Guidelines RAG Assistant

A retrieval-augmented generation (RAG) system that answers plain-language questions about clinical guidelines and patient records, with every answer grounded in and cited back to the source document.

## The problem

Clinicians, patients, and caregivers often need quick, accurate answers to specific questions (medication history, condition background, care recommendations) without manually searching through lengthy or highly technical medical records. This project explores how a retrieval-augmented system can surface the right information from clinical text and general medical reference material, then return an answer with a traceable source, rather than a free-form, unverifiable response.

## Data

This project combines two complementary data sources:

1. **Synthetic patient records**, generated by Synthea (synthetichealth.github.io/synthea), in CCDA format. No real patient data is used at any point in this project. Synthea generates realistic but entirely fictional patient histories, which makes it possible to build and demonstrate a healthcare AI project without any HIPAA or privacy concerns.
2. **General medical reference content** from MedlinePlus (medlineplus.gov), the U.S. National Library of Medicine's free consumer health information service. Each patient's diagnosed conditions (via SNOMED CT codes) are matched to a corresponding MedlinePlus topic page using the MedlinePlus Connect API, providing plain-language background on causes, symptoms, and treatment for the conditions that appear in the patient data.

Combining these two sources mirrors how real discharge instructions work: a patient-specific chart paired with general reference material relevant to that patient's diagnoses.

## Pipeline

1. `clean_ccda.py`: parses raw CCDA XML files and extracts clean, section-organized text (Problems, Medications, Allergies, and so on) for each patient
2. `fetch_medline_topics.py`: extracts unique SNOMED CT codes from patient records and pulls matching plain-language topic summaries from MedlinePlus Connect
3. `chunk_documents.py`: splits both patient records and MedlinePlus topics into embedding-ready chunks, tagged by source type
4. `build_vector_store.py`: embeds every chunk (using all-MiniLM-L6-v2) and stores them in a local Chroma vector database
5. `test_retrieval.py`: lets you query the vector store and inspect the top matching chunks from either source
6. (planned) LLM-based question answering with source citation
7. (planned) Evaluation against a hand-built test set of question and answer pairs

## Setup

```bash
git clone https://github.com/xacosta10/clinical-guidelines-rag-assistant.git
cd clinical-guidelines-rag-assistant
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## Usage

Place CCDA XML files in `ccda_raw/`, then run the full pipeline in order:

```bash
python3 clean_ccda.py
python3 fetch_medline_topics.py
python3 chunk_documents.py
python3 build_vector_store.py
python3 test_retrieval.py
```

Cleaned patient text is written to `ccda_cleaned/`, matched MedlinePlus topics are written to `medline_topics/`, and the combined, chunked dataset is stored in `chunks.json` before being embedded into `chroma_db/`.
