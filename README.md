# Clinical Guidelines RAG Assistant

A retrieval-augmented generation (RAG) system that answers plain-language questions about clinical guidelines and patient records, with every answer grounded in and cited back to the source document.

## The problem

Clinicians and care teams often need quick, accurate answers to specific questions (medication history, discharge instructions, guideline recommendations) without manually searching through lengthy documents. This project explores how a retrieval-augmented system can surface the right information from clinical text and return an answer with a traceable source, rather than a free-form, unverifiable response.

## Data

This project uses synthetic patient records generated by Synthea (synthetichealth.github.io/synthea), in CCDA format. No real patient data is used at any point in this project. Synthea generates realistic but entirely fictional patient histories, which makes it possible to build and demonstrate a healthcare AI project without any HIPAA or privacy concerns.

## Pipeline

1. `clean_ccda.py`: parses raw CCDA XML files and extracts clean, section-organized text (Problems, Medications, Allergies, and so on) for each patient, ready for chunking and embedding
2. (in progress) Embedding and vector storage
3. (planned) Retrieval and LLM-based question answering with source citation
4. (planned) Evaluation against a hand-built test set of question and answer pairs

## Setup

\`\`\`bash
git clone https://github.com/xacosta10/clinical-guidelines-rag-assistant.git
cd clinical-guidelines-rag-assistant
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
\`\`\`

## Usage

Place CCDA XML files in `ccda_raw/`, then run:

\`\`\`bash
python3 clean_ccda.py
\`\`\`

Cleaned text files are written to `ccda_cleaned/`.
